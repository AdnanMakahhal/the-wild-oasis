======================================================
ملخص مشروع The Wild Oasis للدراسة والمراجعة (React.js)
======================================================

هذا الملف يحتوي على شرح شامل لكل التقنيات والمكتبات والمفاهيم المميزة التي تم استخدامها في هذا المشروع لتكون مرجعاً ممتازاً لك في المراجعة والدراسة.

------------------------------------------------------
1. التقنيات الأساسية (Core Technologies):
------------------------------------------------------
- React (v18): مكتبة الواجهات الأساسية (UI Library).
- Vite: أداة بناء المشروع (Build Tool) وهي أسرع بكثير من Create React App (CRA).

------------------------------------------------------
2. جلب البيانات وإدارة الحالة (Data Fetching & State Management):
------------------------------------------------------
- React Query (@tanstack/react-query): 
  من أهم المكتبات في المشروع! تم استخدامها لجلب البيانات (Fetching) من 서버 (Supabase)، وتخزينها محلياً (Caching)، وتحديثها (Mutating).
  * المفاهيم: useQuery (لجلب البيانات) و useMutation (للتعديل/الإضافة/الحذف).
- Context API:
  مستخدم لإدارة الحالة العامة (Global State) للتطبيق مثل الـ Dark Mode (الوضع الليلي) من خلال `DarkModeContext`.
- React State (useState, useEffect):
  لإدارة الحالة المحلية داخل المكونات (Local UI State).

------------------------------------------------------
3. توجيه الصفحات (Routing):
------------------------------------------------------
- React Router DOM (v6):
  تم استخدامه للتنقل بين الصفحات في التطبيق (Single Page Application - SPA).
  * المفاهيم: 
    - `<BrowserRouter>` و `<Routes>` و `<Route>`.
    - الصفحات المحمية (Protected Routes) لمنع دخول المستخدمين غير المسجلين لصفحة الـ Dashboard.
    - AppLayout (التخطيط الموحد) الذي يشمل الشريط الجانبي (Sidebar) والهيدر، ويتغير فقط المحتوى الداخلي باستخدام `<Outlet>`.

------------------------------------------------------
4. تصميم الواجهات وتنسيقها (Styling):
------------------------------------------------------
- Styled Components:
  مكتبة CSS-in-JS تتيح لك كتابة CSS داخل ملفات الجافاسكريبت. تساعد في عزل الأكواد وعدم تداخلها.
  * المفاهيم: `GlobalStyles` لتعريف المتغيرات (CSS Variables) والخطوط الأساسية.

------------------------------------------------------
5. التعامل مع النماذج (Forms):
------------------------------------------------------
- React Hook Form:
  مكتبة رائعة لإدارة النماذج (Forms) وعمل التحقق من صحة البيانات (Validation) بأقل كمية من إعادة التصيير (Re-renders).

------------------------------------------------------
6. قواعد البيانات والباك إند (Backend as a Service):
------------------------------------------------------
- Supabase:
  تم استخدامه كبديل لـ Firebase لإنشاء قواعد البيانات، وتوفير الـ API، ونظام تسجيل الدخول (Authentication) والمصادقة، ورفع الملفات (Storage) مثل صور الكبائن.

------------------------------------------------------
7. مكتبات مساعدة أخرى (Helpers & Utilities):
------------------------------------------------------
- React Hot Toast: لإظهار إشعارات (Notifications/Toasts) للمستخدم (مثل: تم الحفظ بنجاح، أو خطأ في تسجيل الدخول).
- Recharts: مكتبة مخصصة لإنشاء الرسوم البيانية (Charts) الموجودة في صفحة الـ Dashboard.
- date-fns: مكتبة قوية للتعامل مع التواريخ والأوقات (حساب فرق الأيام بين الحجوزات، تنسيق التواريخ، إلخ).
- React Icons: لتوفير الأيقونات المستخدمة في المشروع بسهولة.
- React Error Boundary: للتعاطي مع الأخطاء غير المتوقعة (Crashes) وعرض صفحة خطأ بدلاً من الصفحة البيضاء (Fallback UI).

------------------------------------------------------
8. مفاهيم React وأنماط التصميم (React Patterns):
------------------------------------------------------
- Custom Hooks (الهوكس المخصصة):
  تمت كتابة Hooks مخصصة (مثل: `useLogin`، `useDarkMode`، `useLocalStorageState`) لعزل اللوجيك عن واجهة المستخدم (UI) مما يجعل الكود نظيفاً وسهل الصيانة.
- Render Props / Compound Components:
  وهي أنماط تصميم متقدمة في React يتم استخدامها لبناء مكونات مرنة جداً مثل الـ Modal (النوافذ المنبثقة) وقوائم السياق (Menus).
- Separation of Concerns (فصل الاهتمامات):
  لاحظ كيف تم فصل الـ `features` (ميزات المشروع مثل الحجوزات، الكبائن) عن الـ `ui` (المكونات البصرية المجردة مثل الأزرار والـ Inputs)، وعن الـ `services` (التي تتخاطب مع Supabase).

=== 
نصيحة للمراجعة: ابدأ بملف App.jsx لتفهم مسارات التطبيق وحركة البيانات، ثم انتقل لملف package.json لتذكر المكتبات، وبعدها راجع أحد الـ Features (مثل Cabins أو Bookings) لترى كيف تتكامل هذه التقنيات مع بعضها!
